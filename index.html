So many repos, so little time

If you use github for a decent size organization, chances are you have many repos you keep a check on. In fact, if you've been with the same organization for a year or two, you've probably moved around to another project at least one time if not more.

With so many repos you have to keep track of and want to have the lastest version of you've came up with a script or some way to automate the process.

Well, I have situation as above (we have well over 150 repos) and found myself in a constant state of peforming a 

$ git pull
in way too many repos through any given time.

I hacked a bit

At first I created a simple bash script to automate the process but new repos were always being added and the initial script I created didn't take that into account but that only got me so far and wasn't hacked enough.

I hacked some more

I decied to play with Python and the Requests module.

I came across a number of github modules as well but since I wanted to learn a bit more about the Requests module I decided to stay away from them for this exercise.

End result

The fruits of that labor ended up being a python script, feeding off a config.json file, which will go out for a given organization and perform a git command across all its repos.

I tend to mainly use the 'status' or 'pull' commands.

I've open sourced the code on github under the repo git-repo-batch-command.

The usage is pretty simple

usage: grcb [-v | -h] cmd
Options and arguments:
-v     : show version info
-h     : print help / usage
cmd    : git command to execute across all repos in your organization

And the config file is quite simple as well. In my case it looks something like:

{
  "gituser" : {
    "username" : "bobwilliams",
    "password" : "",
    "organization" : "sparcedge"
  }
}

Conclusion

Of course there is a lot more I would like to do with this so please head on over to the repo and make this little tool way better than it is today :)
